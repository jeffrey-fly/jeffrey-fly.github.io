多核并发与屏障

Posted on 2026-03-18 Views: Disqus:

一 内存屏障概念

屏障（fence）作用：保证 指令顺序和跨线程可见性

  1. 编译器屏障（compiler fence）: 阻止编译器重排指令，不生成 CPU 指令
  2. CPU 内存屏障（CPU fence / hardware fence）：阻止 CPU 乱序执行，确保 store/load 对其他核可见

二 CPU 内存模型对比

架构 内存模型 顺序保证
x86 / x86-64 TSO（Total Store Order） StoreStore / LoadLoad / LoadStore 保证，Store→Load 可能乱序
ARM / AArch64 Weakly Ordered / Relaxed StoreStore / LoadLoad / StoreLoad 都可能乱序，需要显式 fence
  • StoreLoad hazard：前面的 store 在 store buffer 中，后续 load 先看到索引或标志 → 可能读取到旧值
  • 解决方案：release/acquire fence，建立跨线程 happens-before 链

三 Release / Acquire Fence

  • Release store（写端）

    • 发布数据，保证前写在逻辑上“先发生”

  • Acquire load / fence（读端）

    • 获取发布数据，保证读取时看到 release store 之前的写

配对原则

写入端 读取端 作用
release store acquire load/fence 建立跨线程 happens-before 链
普通 store acquire load ❌ 无法保证可见性
release store 普通 load ❌ 读取线程可能看不到前写

跨平台写无锁代码时必须使用 release/acquire，单纯依赖 x86 TSO 的天然顺序不可移植

五 总结

  • x86 TSO：顺序天然保证 + release/acquire 主要用于跨线程可见性
  • ARM / AArch64：顺序不保证，必须 release/acquire 或 fence
